## Обновление операционной системы RouterOS

Рекомендуется использовать последнюю версию RouterOS v7. Обновить MikroTik можно двумя способами: автоматически через Интернет; вручную с помощью файла обновления.

При наличии доступа в Интернет достаточно выполнить автоматическое обновление. Если доступа нет — обновление выполняется вручную. После обновления устройство необходимо перезагрузить.

RouterOS использует пакеты (`.npk`), которые содержат отдельные функции системы. Основной пакет обязателен для работы устройства, дополнительные устанавливаются по необходимости. Версии всех установленных пакетов должны совпадать, иначе возможны сбои. Для разных моделей MikroTik используются разные наборы пакетов.

Перед настройкой рекомендуется обновить RouterOS до актуальной версии.

```fold title:"Update RouterOS"
/system/package/update/
check-for-updates
install
```
---
## Обновление загрузчика RouterBOOT

После обновления RouterOS необходимо обновить загрузчик RouterBOOT, который отвечает за запуск системы при включении устройства. Версии RouterOS и RouterBOOT должны совпадать, иначе возможны сбои в работе устройства, например проблемы с DHCP-сервером и другими функциями.

```fold title:"Update RouterBOOT"
/system/routerboard/
upgrade

/system/
reboot
```
---
## Сброс настроек

Чтобы избежать конфликтов между старыми и новыми настройками, рекомендуется выполнить сброс без применения конфигурации по умолчанию. После сброса устройство появится в WinBox на вкладке Neighbors. Так как IP-адрес будет отсутствовать, подключение нужно выполнять по MAC-адресу.

```fold title:"Reset configuration"
/system/
reset-configuration no-defaults=yes
```
---

## Установка пароля администратора

После сброса настроек необходимо сразу задать пароль для учетной записи `admin`. Это особенно важно для устройств, где пароль по умолчанию отсутствует. После подключения роутера к Интернету устройство может стать целью попыток взлома.

Рекомендуется использовать сложный пароль длиной не менее 8 символов с буквами разного регистра, цифрами и специальными символами.

```fold title:"Set admin password"
/password

Change your password
new password>
```

---
## Назначение имени роутера

Если в сети используется несколько устройств MikroTik, рекомендуется задать имена для удобной идентификации. Лучше использовать короткие и понятные названия, например:
- роутеры — `R1`, `R2` или `GW1`, `GW2`;
- коммутаторы — `SW1`, `SW2`;
- точки доступа — `AP1`, `AP2`.

Для одного устройства имя не обязательно, но его настройка не влияет на работу устройства.

```fold title:"Set device name"
/system/identity/
set name=GW
```

---
## Переименование внешнего интерфейса

Рекомендуется переименовать интерфейс ether1 в ether1-WAN1. Это упрощает настройку и анализ конфигурации, так как назначение интерфейса сразу видно в правилах и настройках, где он используется. Комментарии к интерфейсам менее удобны, поскольку они не отображаются в связанных правилах.

```fold title:"Rename WAN interface"
/interface/ethernet/
set [find default-name=ether1] name=ether1-WAN1
```

---
## Объединение внутренних интерфейсов

Все внутренние интерфейсы следует объединить в один bridge, чтобы они находились в одном L2-сегменте. В данном примере в bridge входят интерфейсы `ether2–ether5`, `wifi1` и `wifi2`. В дальнейшем настройки LAN нужно применять к bridge-интерфейсу, а не к отдельным портам.

```fold title:"Create LAN bridge"
/interface/bridge/
add name=bridge

/interface/bridge/port/
add bridge=bridge interface=ether2
add bridge=bridge interface=ether3
add bridge=bridge interface=ether4
add bridge=bridge interface=ether5
add bridge=bridge interface=wifi1
add bridge=bridge interface=wifi2
```

## Настройка IP-адресации

### Назначение IP-адреса на WAN-интерфейс

Интернет-провайдер может выдавать сетевые настройки тремя способами:
- статически — вручную задаются IP-адрес, маска сети, шлюз и DNS-серверы;
- автоматически через DHCP;
- через PPPoE.

#### Настройка статического IP-адреса

Если провайдер использует статическую адресацию, IP-адрес необходимо назначить вручную. В данном примере используются:
- IP-адрес: `100.64.111.100/24`
- шлюз: `100.64.111.1`

Адрес шлюза будет использоваться позже при настройке маршрута по умолчанию.

```fold title:"Set WAN IP address"
/ip/address/
add address=100.64.111.100/24 interface=ether1-WAN1
```
#### Настройка DHCP-клиента на WAN-интерфейсе

Если провайдер выдает сетевые настройки через DHCP, на интерфейсе ether1-WAN1 нужно создать DHCP-клиент. Обычно вместе с IP-адресом автоматически передаются DNS-серверы и маршрут по умолчанию, поэтому дополнительные настройки не требуются.

```fold title:"Enable DHCP client on WAN"
/ip/dhcp-client/
add disabled=no interface=ether1-WAN1
```
#### Настройка PPPoE-клиента на WAN-интерфейсе

Если провайдер использует PPPoE, на интерфейсе `ether1-WAN1` нужно создать PPPoE-клиент и указать логин и пароль. Обычно вместе с подключением автоматически создаются маршрут по умолчанию и настройки DNS.

```fold title:"Configure PPPoE client"
/interface/pppoe-client/
add add-default-route=yes disabled=no interface=ether1-WAN1 name=pppoe-WAN1 \
   password=password_pppoe service-name=isp_pppoe user=client123
```

### Назначение IP-адреса на LAN-интерфейс

На bridge-интерфейс `bridge-LAN` необходимо назначить IP-адрес локальной сети. В данном примере используется адрес `192.168.100.1/24`.

```fold title:"Set LAN IP address"
/ip/address/
add address=192.168.100.1/24 interface=bridge-LAN
```

---
## Добавление маршрута по умолчанию

После назначения IP-адреса на bridge-LAN рекомендуется подключаться к роутеру по IP-адресу, а не по MAC-адресу, так как такое подключение стабильнее.

Маршрут по умолчанию используется для отправки трафика во внешние сети. Если на WAN-интерфейсе используется статический IP-адрес, маршрут нужно добавить вручную, указав шлюз провайдера. В данном примере используется шлюз 100.64.111.1.

При получении адреса через DHCP или PPPoE маршрут по умолчанию 
обычно создается автоматически.

### Проверка наличия динамического маршрута

Динамический маршрут по умолчанию появляется автоматически только при получении IP-адреса через DHCP или PPPoE.

```fold title:"Check routing table"
/ip/route/
print
Columns: DST-ADDRESS, GATEWAY, DISTANCE
#      DST-ADDRESS       GATEWAY        DISTANCE
  DAd  0.0.0.0/0         100.64.111.1          1
  DAc  100.64.111.0/24   ether1-WAN1           0
  DAc  192.168.100.0/24  bridge-LAN            0
```

### Добавление статического маршрута по умолчанию

Если на WAN-интерфейсе используется статический IP-адрес, маршрут по умолчанию нужно добавить вручную.

```fold title:"Add default route"
/ip/route/
add dst-address=0.0.0.0/0 gateway=100.64.111.1
```

### Проверка подключения к Интернету

После настройки маршрутизации роутер должен успешно выполнять `ping` до внешних IP-адресов, например:
- `1.1.1.1`
- `8.8.8.8`
- `77.88.8.1`
- `77.88.8.8`

Проверку нужно выполнять именно с роутера и только по IP-адресам, а не по доменным именам. Если `ping` не проходит, проверьте настройки или уточните параметры подключения у провайдера.

---
## Настройка DNS-клиента

DNS нужен роутеру для доступа к ресурсам по доменным именам. Без настройки DNS не будут работать обновления RouterOS, синхронизация времени по DNS-именам и другие функции.

Если WAN-интерфейс получает настройки через DHCP, DNS-серверы обычно назначаются автоматически. При необходимости можно указать собственные DNS-серверы — они будут использоваться вместо полученных от провайдера.

В данном примере используются DNS-серверы `77.88.8.8` и `77.88.8.1`.

```fold title:"Set DNS servers"
/ip/dns/
set servers=77.88.8.8,77.88.8.1
```

### Проверка DNS

После настройки DNS роутер должен успешно выполнять `ping` до доменных имен, например:
- `ya.ru`
- `rbc.ru`
- `rg.ru`

Проверку нужно выполнять с самого роутера. Если `ping` по IP-адресам проходит, а по доменным именам — нет, проблема обычно связана с DNS-серверами.

---
## Включение DNS-сервера

Если устройства в локальной сети должны использовать роутер MikroTik как DNS-сервер, необходимо включить обработку DNS-запросов.

Использовать роутер в роли DNS-сервера следует только при необходимости. Например, в сети с контроллером домена клиентские устройства должны использовать DNS-сервер контроллера домена.

После включения роутер начнет принимать DNS-запросы на всех интерфейсах. Поэтому доступ к DNS-серверу рекомендуется ограничить правилами файрвола только для локальной сети.

```fold title:"Enable DNS server"
/ip/dns/
set allow-remote-requests=yes
```

## Настройка DHCP-сервера

DHCP-сервер автоматически выдает устройствам в локальной сети IP-адрес, шлюз и DNS-сервер.

В данной конфигурации используются следующие параметры:

- диапазон IP-адресов: `192.168.100.101–192.168.100.199`;
- шлюз: `192.168.100.1`;
- DNS-сервер: `192.168.100.1`;
- время аренды: `3 дня`.

```fold title:"Configure DHCP server"
/ip/pool/
add name=dhcp-lan-pool ranges=192.168.100.101-192.168.100.199

/ip/dhcp-server/
add address-pool=dhcp-lan-pool interface=bridge-LAN lease-time=3d name=dhcp-lan

/ip/dhcp-server/network/
add address=192.168.100.0/24 dns-server=192.168.100.1 gateway=192.168.100.1
```

### Проверка DHCP

Устройства в локальной сети должны автоматически получать сетевые настройки по DHCP.

После получения адреса должны успешно выполняться:
- `ping` до IP-адреса роутера `192.168.100.1`;
- `ping` до WAN-интерфейса роутера.

Доступ в Интернет на этом этапе может еще не работать.

## Настройка NAT

NAT используется для доступа устройств из локальной сети в Интернет через IP-адрес WAN-интерфейса роутера.

На MikroTik NAT обычно настраивается двумя способами:
- `src-nat` — используется при постоянном внешнем IP-адресе;
- `masquerade` — используется при динамическом внешнем IP-адресе.

### NAT с помощью src-nat

Используйте `src-nat`, если провайдер выдает постоянный IP-адрес.

```fold title:"Configure src-nat"
/ip/firewall/nat/
add action=src-nat chain=srcnat out-interface=ether1-WAN1 to-addresses=100.64.111.100
```

### NAT с помощью masquerade

Используйте `masquerade`, если внешний IP-адрес может изменяться.

```fold title:"Enable DNS server"
/ip/firewall/nat/
add action=masquerade chain=srcnat out-interface=ether1-WAN1
```
### Проверка NAT

После настройки NAT устройства из локальной сети должны получить доступ в Интернет.

Проверьте:
- `ping` до внешних IP-адресов;
- `ping` до доменных имен.

Если `ping` по IP-адресам проходит, а по доменным именам — нет, проблема связана с DNS.

## Настройка Wi-Fi

MikroTik hAP ax2 поддерживает два диапазона Wi-Fi: `2,4 ГГц` и `5 ГГц`.

Для повседневного использования рекомендуется диапазон `5 ГГц`, так как диапазон `2,4 ГГц` обычно перегружен и обеспечивает более низкое качество связи. Если используются оба диапазона, для них лучше задать разные SSID.

В данной конфигурации для сети `2,4 ГГц` используются следующие параметры:
- интерфейс переименован в `wlan-2GHz`;
- SSID: `GW-2`;
- включены только `WPA2-PSK` и `WPA3-PSK`;
- WPS отключен;
- ширина канала: `20 МГц`;
- включены `RTS/CTS`;
- автоматический выбор частоты выполняется каждые `2–4 часа`;
- используется режим `Superchannel`;
- диапазон частот: `2412–2472 МГц`;
- мощность передатчика: `16 dBm`.

```fold title:"Configure 2.4 GHz Wi-Fi"
/interface/wifi/
set [ find default-name=wifi2 ] channel.band=2ghz-ax .frequency=2412-2472 \
   .reselect-interval=2h..4h .width=20mhz configuration.country=Superchannel \
   .hw-protection-mode=rts-cts .mode=ap .ssid=GW-2 .tx-power=16 disabled=no name=wlan-2GHz \
   security.authentication-types=wpa2-psk,wpa3-psk .passphrase=password .wps=disable
```

Для диапазона `5 ГГц` используются следующие параметры:
- интерфейс переименован в `wlan-5GHz`;
- SSID: `GW-5`;
- включены только `WPA2-PSK` и `WPA3-PSK`;
- WPS отключен;
- ширина канала: `40 МГц`;
- включены `RTS/CTS`;
- автоматический выбор частоты выполняется каждые `2–4 часа`;
- используется режим `Superchannel`;
- диапазоны частот: `5180–5240`, `5260–5320`, `5660–5720`, `5745–5825`;
- мощность передатчика: `18 dBm`.

```fold title:"Configure 5 GHz Wi-Fi"
/interface/wifi/
set [ find default-name=wifi1 ] channel.band=5ghz-ax .frequency=\
   5180-5240,5260-5320,5660-5720,5745-5825 .reselect-interval=2h..4h \
   .width=20/40mhz configuration.country=Superchannel .hw-protection-mode=rts-cts .mode=ap .ssid=GW-5 \
   .tx-power=18 disabled=no name=wlan-5GHz \
   security.authentication-types=wpa2-psk,wpa3-psk .passphrase=password .wps=disable
```

## Настройка файрвола

IPv4-файрвол MikroTik использует три цепочки:
- `Input` — трафик к самому роутеру;
- `Output` — трафик от самого роутера;
- `Forward` — транзитный трафик через роутер.

В данной конфигурации настраиваются только цепочки `Input` и `Forward`, так как отдельная настройка `Output` обычно не требуется.

Файрвол будет работать по следующему принципу:
- подключения из Интернета запрещены по умолчанию;
- подключения из локальной сети разрешены по умолчанию.

Для разрешения отдельных сервисов, например VPN, нужно добавлять дополнительные правила. При этом важно учитывать порядок правил внутри цепочек, так как он влияет на обработку трафика.

В цепочке `Input` создаются четыре основных правила:
1. Разрешение `established` и `related` соединений — снижает нагрузку на процессор за счет быстрой обработки уже установленных соединений.
2. Блокировка `invalid` соединений — отбрасывает некорректный трафик.
3. Разрешение `ICMP` — позволяет использовать `ping` для диагностики.
4. Блокировка новых подключений не из локальной сети — запрещает доступ к роутеру из Интернета, но сохраняет полный доступ из локальной сети.

```fold title:"Configure Input chain firewall rules"
/ip/firewall/filter/
add action=accept chain=input connection-state=established,related \
   comment="accept established related" 
add action=drop chain=input connection-state=invalid comment="drop invalid" 
add action=accept chain=input protocol=icmp comment="accept ICMP" 
add action=drop chain=input in-interface=!bridge-LAN \
   comment="drop all from not LAN"
```

В цепочке `Forward` создаются три основных правила:
1. Разрешение `established` и `related` соединений — ускоряет обработку уже установленных соединений и снижает нагрузку на процессор.
2. Блокировка `invalid` соединений — отбрасывает некорректный трафик.
3. Блокировка новых подключений из Интернета — запрещает входящий транзитный трафик с WAN-интерфейса, кроме соединений, разрешенных через `dst-nat` (проброс портов).

```fold title:"Configure Forward chain firewall rules"
/ip/firewall/filter/
add action=accept chain=forward \
   connection-state=established,related \
   comment="accept established & related"
add action=drop chain=forward \
   connection-state=invalid \
   comment="drop invalid"
add action=drop chain=forward \
   connection-nat-state=!dstnat \
   in-interface=ether1-WAN1 \
   comment="drop all from WAN"
```

## Настройка дополнительных параметров безопасности

MikroTik может обнаруживать соседние устройства через `MNDP` и `LLDP`. Чтобы роутер не был виден из недоверенных сетей, обнаружение соседей рекомендуется разрешать только на LAN-интерфейсах.

Также рекомендуется:
- разрешить MAC-доступ к консоли только из локальной сети;
- разрешить MAC-подключение WinBox только из локальной сети;
- отключить `MAC-ping`.

В данной конфигурации доверенным интерфейсом является `bridge-LAN`.

```fold title:"Configure Layer 2 security"
/interface/list/
add name=LAN

/interface/list/member/
add interface=bridge-LAN list=LAN

/ip/neighbor/discovery-settings/
set discover-interface-list=LAN

/tool/mac-server/
set allowed-interface-list=LAN

/tool/mac-server/mac-winbox/
set allowed-interface-list=LAN

/tool/mac-server/ping/
set enabled=no
```

Даже после сброса настроек MikroTik автоматически назначает IPv6-адреса интерфейсам. При этом правила IPv4-файрвола не защищают IPv6-трафик. Если IPv6 не используется, его рекомендуется отключить.

После отключения IPv6 роутер необходимо перезагрузить.

```fold title:"Disable IPv6"
/ipv6/settings/
set disable-ipv6=yes
```

Для повышения безопасности рекомендуется отключить неиспользуемые службы:
- `ftp`
- `telnet`
- `www`
- `api`
- `api-ssl`

Службы `www` и `www-ssl` используются для доступа к WebFig по HTTP и HTTPS.

```fold title:"Disable unused services"
/ip/service/
set ftp disabled=yes
set telnet disabled=yes
set www disabled=yes
set api disabled=yes
set api-ssl disabled=yes
```

Корректное время необходимо для работы сертификатов, расписаний, логов и других функций RouterOS.

Для синхронизации времени нужно:
- включить NTP-клиент;
- указать NTP-сервер;
- вручную задать часовой пояс.

Автоматическое определение часового пояса рекомендуется отключить, так как оно может работать некорректно.

```fold title:"Configure NTP client"
/system/ntp/client/
set enabled=yes

/system/ntp/client/servers/
add address=ru.pool.ntp.org

/system/clock/
set time-zone-autodetect=no time-zone-name=Europe/Moscow
```
